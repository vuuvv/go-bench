# Go Plus 代码注释规范

## 目标

Go Plus 的核心逻辑会处理 Go AST、源码位置、`go test -run` 正则路径和 VSCode Extension Host 生命周期。注释用于解释模块职责、关键算法、边界条件和设计取舍，帮助维护者判断某个分支为什么存在。

## 基本要求

- 注释默认使用中文，API 名称、配置键、命令和代码标识符保留英文。
- 每个公开导出的函数、类型、接口、命令和配置项必须有用途说明。
- parser、detector、locator、runner、codelens 等核心模块入口必须有模块级注释。
- 对动态 case name、不完整源码、无法确认映射关系等跳过分支，必须说明跳过原因。
- 正则转义、Go test run pattern 构造、路径解析、debounce、缓存和 child process 调用必须解释边界条件。
- 测试 fixture 中用于覆盖特殊场景的代码应说明该 fixture 覆盖的模式。

## 避免的问题

- 避免只复述代码，例如“把值赋给变量”。
- 修改逻辑时必须同步更新注释，不能留下与实现不一致的说明。
- 不为显而易见的局部变量添加噪声注释。
